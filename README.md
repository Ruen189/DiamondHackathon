# Приложение для автоматизации документооборота отдела улучшений.

Команда: Даймонд

Название проекта: Алмаз

## Что реализовано

- Единое клиент-серверное приложение (FastAPI + встроенный web-клиент).
- Авторизация и регистрация (логин/email/телефон как `identifier`), роли `admin/user`.
- Окна в левой панели:
  - `Окно анализа` (аудио, загрузка файла в базу знаний, импорт Jira SM, поиск похожих кейсов, выжимка идей через LLM).
  - `Окно документы` (база документов по папкам, открытие, редактирование, применение шаблонов 6С/СОП/SMED/TPM).
  - `Окно нейронки` (чат с агентом + метрики времени/скорости генерации для каждого промпта).
  - `Настройки` (профиль, KPI из ТЗ, у `admin` блок "Для разработчиков").
- В `admin`-блоке:
  - выбор режима запуска моделей: `server` (по умолчанию) / `local` / `api`;
  - проверка статуса модельного сервера;
  - запуск сервера через `docker compose up -d` с проверкой, что он не запущен.
- RAG (MVP): индексирование текстовых документов в чанки + поиск похожих фрагментов.

## Стек

- Backend: FastAPI + SQLite
- Frontend: HTML/CSS/JS (стиль основан на `interface.html`)
- Модели:
  - LLM: `qwen-3.5-4b-q5_k_m.gguf` через `llama-server`
  - Audio: `whisper.cpp large-v3-turbo-q8_0`

## Переменные окружения

Используйте `.env` на основе `.env.example`:

- `PORT`
- `TEXT_MODEL`
- `AUDIO_MODEL`
- `ADMIN_LOGIN`
- `ADMIN_PASSWORD`
- `CONTEXT_WINDOW` (флаг `-c`, по умолчанию `200000`)
- `JWT_SECRET`
- `LLAMA_SERVER_URL`
- `WHISPER_SERVER_URL`

## Запуск

1. Установить зависимости:
   - `python -m venv .venv`
   - `.venv\\Scripts\\activate`
   - `pip install -r requirements.txt`
2. Создать `.env` (можно копией `.env.example`).
3. Запустить приложение:
   - `python -m app.main`
4. Открыть: `http://localhost:8000` (или ваш `PORT`).

## Модельный сервер

`docker-compose.yml` содержит:

- `llama-server` (порт `8080`)
- `whisper-server` (порт `8081`)

Запуск можно делать вручную (`docker compose up -d`) или из интерфейса в настройках под `admin`.
